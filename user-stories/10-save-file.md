# User Story: Save File to User Space

## Prerequisites
- Files browser implemented (from 07-files-browser)
- S3Provider and LocalFileSystemProvider implemented (see @specs/BACKEND.md)
- New chat functionality implemented (from 08-new-chat)
- User must be authenticated (login flow from 02-login)
- AuthenticatedUser extractor implemented (see @specs/AUTH-MIDDLEWARE.md)

## Flow
```
ChatMessage → Assistant generates file → Assistant calls SaveFile tool → File saved, link returned
```

---

**Tool:** `SaveFile`  
**Called by:** Assistant (during chat conversation)

**Parameters:**
| Parameter | Type | Description |
|-----------|------|-------------|
| path | string | File path like `/project/file_name.md` |
| content | string | File content generated by Assistant |
| tags | array | Tags for the document, given by AI |

**Tool Call Example:**
```json
{
  "tool": "SaveFile",
  "arguments": {
    "path": "/project/poem.txt",
    "content": "Roses are red\nViolets are blue\nPoetry is fun\nFor me and you",
    "tags": ["poem", "creative"]
  }
}
```

**Note:** `root_document` (link to current chat) is automatically added by the tool based on context.

**Response:**
```json
{
  "status": "ok",
  "file": {
    "id": "uuid",
    "path": "/project/poem.txt",
    "link": "file://uuid"
  }
}
```

---

**UI:** User sees a clickable link in chat that opens the saved file content.

---

## Acceptance Criteria

### Backend
- [ ] `SaveFile` tool is available to the LLM during chat — accepts `path`, `content`, `tags`, stores file in S3 under `{user_id}/{path}`
- [ ] Tool automatically links saved file to the current chat via `root_document`
- [ ] File metadata (id, path, tags, root_document) is persisted in SQLite
- [ ] Integration tests cover: save file via tool, file appears in S3, metadata stored in SQLite, file linked to chat, duplicate path handling
- [ ] `cargo test` — all tests pass, zero failures
- [ ] Backend starts with config file, serves HTTP on configured port
- [ ] `docker-compose.yml` includes backend, frontend, MinIO, and required dependencies

### Frontend
- [ ] Chat messages containing `file://uuid` links render as clickable links
- [ ] Clicking a file link opens the saved file content (navigates to file view or displays inline)
- [ ] Frontend unit tests pass — file link detection and rendering in chat messages
